Statyczny język vs Dynamiczny

Statyczny język programowania
- Sprawdzanie kodu podczas kompilowania ~Compile Time
- Potrzeba zdeklarowania zmiennych
- Bardziej bezpieczny

Dynamiczny język programowania
- Kod jest sprawdzany podczas czasu uruchomionego programu ~Run Time
- Nie ma potrzebny deklaracji zmiennych
Na przykład gdy mamy sytuacje; Stwierdziliśmy, że potrzebna nam funkcja która polega na zapisaniu nowej zmiennej int poprzez wprowadzenie danych przez użytkownika. Użytkownik podczas programu wpisał, złą wartość String zatem otrzymaliśmy error podczas programu. 
- Jest szybszy w pisaniu, ponieważ możemy używać kodu wiele razy, ale może sprawdziać problemy poprzez błędy in run time


Remember about flowcharts

Flowcharts its primiary way to show your idea/concept of your code.
The best 2 ways are psuedocode and blockcharts

Psuedocode look something like this:

start
input num
c = 2
while c != num:
if c < 10
output "is less than 10"
exit
c = c + 1
end while
output "c is equal to num"
exit
